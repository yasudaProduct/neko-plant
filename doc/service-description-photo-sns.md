# neko-plant サービス説明（植物×猫 フォトSNS コンセプト）

## 1. サービス概要

**neko-plant** は、日本の家庭における「猫と植物の暮らし」を共有し、あわせて植物と猫の共存に関する情報を蓄積するプラットフォームです。

**写真を中心とした SNS 的な体験**と、植物カタログ・検索を組み合わせます。ユーザーは猫と植物が一緒に写った写真を投稿し、植物名は AI 判定または手動でタグ付けします。投稿が増えるほど、「どの植物が、どれだけ多くの猫との共存実績として観測されているか」が可視化されます。

---

## 2. コアコンセプト

### 2.1 一言でいうと

**「猫と植物の暮らしを共有する」** — Instagram に近い写真中心のフィードと、植物カタログ・検索を組み合わせた体験です。

### 2.2 サービスの特徴

| 観点 | 内容 |
|------|------|
| 主な行動 | フィードを眺め、写真を投稿する |
| 安全性の表現 | **共存実績の多さ**（ポジティブリスト） |
| 投稿の起点 | 写真を撮ってそのまま投稿 |
| コンテンツの主役 | **猫と植物の写真** |
| 全体像 | **SNS フィードを中心としたサービス** |

「この植物と猫が一緒に暮らしている」という**投稿そのもの**が、安全性に関する**暗黙のエビデンス**として蓄積されます。明示的な good／bad 評価に依存せず、投稿の分布から読み取れる設計です。

---

## 3. 想定ユーザーと動機

| ユーザー像 | 主な動機 | 典型的な行動 |
|-----------|---------|-------------|
| 猫と植物の両方を飼育している人 | かわいい写真を共有したい | 投稿する |
| 癒しを求める人 | 猫と植物の写真を見たい | フィードを閲覧する |
| 新しい植物の安全性を調べたい人 | 購入や置き場所の判断材料が欲しい | 植物名で検索し、共存実績を確認する |

写真を楽しむ層と、安全性を確認したい層の両方に届けつつ、安全性に関する情報は**投稿の副産物**として自然に集まることを目指します。

## 4. ユーザーの動線（フロー）

訪れた**動機**ごとに、サイト上で想定する**操作の流れ**を整理する。未実装の機能も、コンセプト上の動線として含めてよい。

### 4.1 サイトに訪れる動機の整理

| ID | 動機（要約） | 説明 |
|----|-------------|------|
| M1 | **写真を楽しみたい** | 猫と植物の写真を眺めて癒やされたい、流し見・暇つぶしとして楽しみたい。 |
| M2 | **暮らしを共有・残したい** | うちの猫と植物が一緒に写った写真を投稿し、記録や共有の場にしたい。 |
| M3 | **植物と猫の共存を調べたい** | これから買う植物、すでに飼っている植物が、猫との同居で問題になりそうか判断材料が欲しい。 |
| M4 | **条件を決めて探したい** | 植物名だけでなく、猫種やタグ、並び順（共存実績の多い順など）で投稿や植物を探したい。 |
| M5 | **アカウントや猫の情報を整えたい** | サインイン、表示名やアイコン、飼い猫プロフィールの登録・編集をしたい。 |
| M6 | **反応したり、人や投稿を辿りたい** | いいねを付けたり、気になる投稿者のプロフィールや投稿一覧を見たりしたい。 |
| M7 | **サービスを初めて知り、把握したい** | 何ができるサイトか、雰囲気や信頼感を掴むために最初の数画面を見たい。 |

M1・M2・M3 は §3 のユーザー像と対応している。M4〜M7 は利用が進むと他の動機と重なりあう。

### 4.2 動機別の操作フロー

#### M1 — 写真を楽しみたい

1. **トップ／フィード**に入る（ブックマーク、検索経由、SNS のリンクなど）。
2. **フィードを縦にスクロール**し、写真・植物タグ・猫の表示を眺める。
3. 気になる場合は **植物タグ** をタップして **植物ページ** へ、**投稿者** をタップして **ユーザープロフィール** へ移動する（**投稿詳細**・拡大表示がある場合はそこからでも同様）。
4. **いいね**を付ける（未ログイン時はサインインへ誘導する想定もありうる）。
5. **フィードへ戻り**、閲覧を続ける。

#### M2 — 暮らしを共有・残したい

1. **サインイン**する（未登録ならアカウント作成）。
2. 初回や猫が未登録の場合、**猫プロフィールを登録**する（M5 と重なる）。
3. **投稿フロー**を開く（フィード上のボタン、ナビなど）。
4. **写真を選択**する（複数枚対応であれば追加）。
5. **植物を紐付ける**（AI 植物判定の候補から選ぶ、または手動検索・タグ付け）。
6. 写っている **猫を選択**する。
7. **コメント**を書く（任意）。
8. **投稿を確定**する。モデレーション待ちの仕様であれば、公開状態の説明を読む。
9. **自分のプロフィール**や **フィード**で、投稿が載ったことを確認する。

#### M3 — 植物と猫の共存を調べたい

1. **植物検索**で名前や通称を入力する、または **フィード** の植物タグ・一覧から目的の植物へ近づく。
2. **植物ページ**で、**学名・分類などのカタログ情報**と **共存実績のサマリー**（何匹の猫との投稿があるか等）を読む。
3. **関連投稿ギャラリー**で写真を確認する。**投稿詳細**があれば拡大して見る。
4. 必要なら **注意喚起**（投稿がない・少ない場合）や、将来想定の **外部毒性データとの照合表示** を読む。
5. 迷いが残る場合は **別の植物ページ** を開く、または **検索結果を並べ替え**（共存実績順など）して比較する。

#### M4 — 条件を決めて探したい

1. **検索・探索**へ移る（または **フィードの絞り込み** があればそこから）。
2. **植物名・タグ**で絞り込む、**猫種**で絞り込む、**並び順**（新着、共存実績の多い植物順など）を変える（未実装の条件は将来の動線として記載）。
3. **結果一覧**から **投稿** または **植物ページ** を開く。
4. **いいね**、**プロフィール閲覧**、**条件を変えて再検索** などへ進む。

#### M5 — アカウントや猫の情報を整えたい

1. **サインイン**する（初回は OAuth 等の導線）。
2. **アカウント設定**で表示名・アイコン・公開範囲などを編集する（項目はプロダクト次第）。
3. **猫プロフィール**を追加・編集する（名前、種類、写真など）。
4. **自分の投稿一覧**や **投稿から集計された植物コレクション**を確認する。
5. **サインアウト**や **退会**が必要なら設定から行う（未実装でも動線として記載可）。

#### M6 — 反応したり、人や投稿を辿りたい

1. **フィード**または **植物ページのギャラリー**で **いいね**を付ける／外す。
2. **投稿者名・アイコン**から **ユーザープロフィール**へ移動する。
3. その人の **投稿一覧**、**猫の紹介**、**一緒に暮らしている植物の一覧**（自動集計）を見る。
4. 特定の投稿から **植物ページ** へ戻る、**フィード**へ戻る。

#### M7 — サービスを初めて知り、把握したい

1. **ランディング**や **フィードの先頭**で、サービスの一言説明や **写真の並び** で雰囲気を掴む。
2. **数件スクロール**するか、**検索**・**植物の例**があればタップして中身を覗く。
3. 続ける場合は **M1**、植物を調べたいなら **M3**、参加したいなら **サインインして M2** へ分岐する。

## 5. 安全性情報の考え方

### 5.1 ポジティブリスト方式

「多くの猫との共存が報告されている植物」は、**観測上、多くの家庭で共存実績がある**ことを示します。一方、投稿が少ない・ない植物は、「猫との共存についてコミュニティからの情報が少ない」状態として扱います。

ユーザー向けの表示イメージは次のとおりです。

- 「〇〇: 234 匹の猫と暮らしています 🐱🌿」
- 「△△: 投稿がありません ⚠️ 猫との共存情報がないため注意してください」

### 5.2 設計上の原則

1. **「危険」と断定しない** — 情報不足は「情報がない」と表現し、法的リスクに配慮します。
2. **外部データとの連携余地** — ASPCA 等の公的データと照合し、毒性が確認されている植物には将来的に外部情報として警告を出す拡張が想定されます。
3. **指標の重み付け** — 共存実績は、可能な範囲で**異なる猫（例: ユニークな猫プロフィール）との実績**を重視し、同一ユーザーの重複投稿だけで水増しされないよう設計します（詳細は実装・データ設計で定義）。

### 5.3 表示ランクの考え方（初期案）

共存に関与した猫の数などに応じて、文言や強調を変えます（閾値は運用データに応じて調整）。

| 共存猫数（目安） | ユーザーへの見せ方のイメージ |
|-----------------|------------------------------|
| 多い（例: 50 以上） | 「多くの猫と暮らしています」など |
| 中程度（例: 10〜49） | 「〇〇匹の猫と暮らしています」 |
| 少数（例: 1〜9） | 「少数の暮らしが報告されています」など |
| 0 | 「猫との共存情報がありません」＋注意喚起 |
